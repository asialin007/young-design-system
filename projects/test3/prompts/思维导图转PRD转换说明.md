# 思维导图转 PRD 转换说明

你将收到一份"产品功能设计思维导图"的 Markdown 内容。请将其完整、无遗漏地转换为结构化的产品功能描述文档(PRD),并严格采用 Markdown 格式输出。

## 转换目标

- 将输入的 Markdown 思维导图转换为结构化的 PRD
- 保留思维导图中的所有功能点与细节,不做主观发挥或删减
- 全文采用统一且规范的标题、分节与条目格式,风格与组织方式严格对齐示例文档

## 输入文档解析规则

### 1. 文档结构识别

- 输入为"思维导图导出为 Markdown"的文本,包含:顶层主题(`#`)、模块(`##`)、分支(`###`)以及以`-`或数字序号开头的层级列表
- `#` 对应文档主标题与产品/平台总主题
- `##` 对应一级模块,如"首页"、"个人中心"、"购物车"等
- `###` 对应模块内的子主题,如"商品详情"、"设置"、"商品列表"、"搜索列表"、"档期列表"、"品牌列表"、"大促会场"、"收藏列表"等
- 列表项代表更细粒度的功能点、字段、操作、文案或交互说明;其缩进层级表示从属关系与上下文归属

### 2. 术语映射规则

- "说明" → 映射为`### 说明`小节,用于概述该页面或功能的作用与使用场景
- "按钮" → 映射为所在页面的"按钮"条目;若涉及弹窗/浮层,则创建`XXX(弹窗/浮层)`小节,记录触发方式、文案、按钮与行为
- "商品列表"/"展示内容" → 映射为"展示内容/字段"条目;子项为具体展示字段,如`商品图片`、`商品名称`、`价格`、`促销标签`等
- "筛选条件"/"搜索" → 映射为"筛选/搜索条件"条目;子项为具体筛选项,如`分类`、`品牌`、`价格区间`、`销量排序`等
- "操作" → 映射为"用户操作"或"操作按钮"条目;常见操作包括`加入购物车`、`立即购买`、`收藏`、`分享`、`查看详情`等
- "提示文案"/"弹窗提示" → 原文保留,归入对应弹窗/提示的小节下;按钮名如`取消`、`确认`、`去购物车`、`继续逛逛`以反引号包裹
- "复选框"/开关/状态 → 归入"商品属性/展示字段/用户选择/状态标识"等位置,如`已售罄`、`促销中`、`新品`、`精选`等

### 3. 结构重组规则

- 若同一模块包含多个页面主题(如"商品列表"、"搜索列表"、"商品详情"、"购物车"、"个人中心"),在 PRD 中分别以独立`###`小节呈现
- 若"按钮"或"操作"下包含"弹窗/浮层/确认提示",在 PRD 中增加对应`XXX(弹窗/浮层)`小节,并展开"触发方式"、"按钮"、"提示文案"、"行为"、"关闭方式"
- 若输入中包含示例(如商品展示示例、价格格式、促销文案模板),保持原位置或提升至最贴近的配置小节中,确保上下文一致

### 4. 内容处理原则

- 严格遵循输入的出现顺序自上而下展开
- 保留全部节点与文本,不合并、不省略、不改义
- 保留原有中英文术语、大小写与缩写
- 将字段/按钮/状态/页面名统一以反引号包裹
- 将数字序号列表规范为无序列表或保留原序号,但不改变语义

## 输出格式规范

### 标题层级

- 文档标题使用`#`
- 模块级标题使用`##`,格式为:`## 【模块名】产品页面功能描述及交互描述`
- 具体页面或子主题使用`###`,例如:`### 说明`、`### 列表页`、`### 新建 XXX 页面`、`### XXX(抽屉/弹窗)`、`### 详情页`、`### 预览(弹窗)`等

### 格式细节

- 在主要模块之间使用`---`分隔,以提升可读性
- 统一使用`-`作为无序列表的前缀
- 保持条目简洁且为单行,必要时再行分解为子条目
- 凡字段名、按钮名、状态名、页面名、示例值、固定术语等,一律使用反引号包裹

### 一致性保证

- 保留思维导图中的原有中文/英文术语与结构,不更名、不合并、不杜撰
- 严格按照输入思维导图的结构与顺序展开为 PRD,层级对应、从上至下展开
